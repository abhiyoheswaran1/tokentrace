# 0.12.0 Final Polish Design

## Goal

Make the 0.12.0 dashboard feel calmer and harder to misunderstand without adding another major capability.

## User Problems

- Route changes can feel frozen on large local data.
- Settings is long enough that users lose the scan controls and export sections.
- Scan results disappear after reload, so users cannot tell what the last scan did.
- Overview totals need a quiet trust checkpoint close to the accounting numbers.
- Evidence is contextual, but direct evidence URLs do not show where the user came from.
- Action labels should describe the action, not the destination noun.
- Empty, partial, and loading states should answer what is happening and what to do next.

## Product Decisions

- Add a subtle global route progress bar for client navigation.
- Add sticky Settings section chips for Scan Controls, Custom Folders, Import Profiles, Guardrails, Package Trust, and Exports.
- Show a persisted Last scan result from scan history when no new scan result exists.
- Add a compact Overview trust footer with latest scan, package IOC, model-rate freshness, and evidence-pack status.
- Add Evidence breadcrumbs and safe return behavior using `from` and `returnTo` query context.
- Standardize user-facing action labels: Scan now, Open Scan Health, Open repair, Set model rate, View evidence, Export pack.
- Keep loading and empty states concise, with next-step actions placed close to the state.

## Non-Goals

- No release, tag, push, or npm publish in this polish pass.
- No new data source adapters beyond the existing 0.12.0 scope.
- No separate Evidence sidebar item; Evidence stays a contextual drill-down.
